## 1. 브랜드 아이덴티티

### 1-1. 브랜드 포지셔닝

```
ScholAI = Scholar(학자) + AI
"초보 연구자의 첫 학술 여정을 함께하는 지식 파트너"

키워드: 신뢰 · 명료함 · 접근성 · 지적 성장
```

### 1-2. 브랜드 성격 (Brand Personality)

| 속성 | 방향 | 피해야 할 것 |
|------|------|------------|
| 톤 | 친절하고 전문적인 멘토 | 딱딱한 학술 기관 느낌 |
| 분위기 | 밝고 집중력 있는 연구 공간 | 과도하게 캐주얼하거나 유치한 느낌 |
| 시각 | 깔끔하고 정보가 명확하게 보이는 | 복잡하고 어수선한 레이아웃 |
| 감성 | 연구가 즐거운 경험이 될 수 있다는 느낌 | 논문이 어렵고 무겁다는 느낌 |

### 1-3. 로고 방향성

```
형태:  "S" + AI 뉴런/회로 이미지 조합 또는
       책 + 빛 번짐 심볼마크
서체:  세련된 산세리프 (Pretendard 또는 Inter)
색상:  딥 네이비 메인 + 민트/아쿠아 포인트
느낌:  학술적이지만 현대적이고 친근한
```

### 2-3. 간격 시스템

```
기본 단위: 4px

16px → 카드 내부 패딩 기본
24px → 섹션 내 요소 간격
32px → 카드 간격
48px → 섹션 간격
64px → 페이지 상단 여백
```

### 2-4. 모서리 & 그림자

```
배지/태그:  4px
카드/버튼:  8px
모달/패널:  12~16px
아바타:     9999px (완전 원형)

카드 기본 그림자: 0 4px 6px -1px rgba(0,0,0,0.1)
카드 호버 그림자: 0 10px 15px -3px rgba(0,0,0,0.2)
모달 그림자:     0 25px 50px -12px rgba(0,0,0,0.5)

## 4. 화면별 UX 가이드 (8화면)

### 화면 1 — 간편 로그인

```
디자인 방향:
  - 전체 화면 다크 배경 (Primary Navy)
  - 중앙 정렬 로그인 카드 (Secondary Navy, 모서리 16px)
  - 로고 상단 배치
  - 소셜 버튼 크게 (최소 48px 높이)
  - 별도 회원가입 링크 없음

버튼:
  - Google: 흰 배경 + 검은 텍스트 + 구글 로고
  - Kakao: #FEE500 배경 + 검은 텍스트 + 카카오 로고

UX 포인트:
  - 로그인 후 페이드 인 → 대시보드 이동
  - 첫 로그인 안내 툴팁 (우측 하단)
```

### 화면 2 — 내 생각이 연구로

```
디자인 방향:
  - 2단 레이아웃: 좌(60% 채팅) / 우(40% rq-ai iFrame)
  - 채팅 패널: 다크 배경, 말풍선 스타일
  - rq-ai 패널: 흰 배경 iFrame, 둥근 테두리
  - 상단 CTA: "이 주제로 선행연구 찾기" (Mint 버튼)

메시지 스타일:
  - 사용자: 우측 정렬, Mint 배경
  - AI: 좌측 정렬, 카드 배경 + AI 아바타
  - 타이핑: 점 3개 펄스 애니메이션

UX 포인트:
  - 엔터키 전송 지원
  - 키워드 추출 후 화면4로 자동 이동 안내 토스트
```

### 화면 3 — 학술지 무엇이 필요할까?

```
디자인 방향:
  - 상단: 5단계 스텝 탭 (① ~ ⑤)
  - 활성 탭: Mint 포인트 + 밑줄
  - 완료 단계: 초록 체크 아이콘
  - 우측: 4인 전문가 멘토 패널 (접이식)
  - 진행률 표시바 상단

4인 멘토 패널 UI:
  - 각 전문가 아바타 + 역할명 배지
  - 선택 전문가 클릭 → 해당 역할 AI 응답
  - 색상 구분:
    · 논문 박사: 보라(#6366F1)
    · 심사위원: 주황(#F59E0B)
    · 지도 교수: 초록(#10B981)
    · 데이터 분석가: 파랑(#3B82F6)

체크리스트:
  - 체크박스 항목 (로컬 저장)
  - 완료 시 취소선 + 초록 체크
```

### 화면 4 — 선행연구 & 연구모형

```
[검색 UI]
  - 키워드 태그 입력창 (엔터로 추가, X로 삭제)
  - 검색 소스 토글: [KCI DB] [국회도서관*] [Google Scholar]
      (* 국회도서관 API 실패 시 국립중앙도서관으로 자동 전환, UI에 안내 표시)
  - 필터 바: 연도 슬라이더 / 등급 필터(KCI/SCI)
  - 결과 카드:
      - 제목 (굵게, 18px)
      - 저자 + 발행연도 + 출처 배지 (KCI DB / 국회도서관 / Scholar)
      - 피인용 횟수: 숫자 + 파란 막대
      - KCI 배지: 보라색 pill
      - SCI/SCIE/SSCI 배지: 금색 pill
      - AI 요약: 연한 배경 박스
      - 버튼: "원문 보기" + "저장"

[연구모형 갤러리]
  - 4열 그리드, 이미지 호버: 스케일 1.02
  - 카드 하단: 논문명(말줄임) + 연도
  - 클릭: 모달 확대 + 배경 딤

UX 포인트:
  - 소스별 로딩 상태 개별 표시 (KCI 빠름, Scholar 느림)
  - 국회도서관 fallback 발생 시: "국립중앙도서관으로 전환 중..." 인라인 알림 표시
  - API 전체 실패 시: 에러 배지 표시 + "KCI DB 결과만 표시 중" 안내
  - 검색 결과 없음: undraw 일러스트 + 안내
```

### 화면 5 — 해외 학술지 Split-View

```
디자인 방향:

### 화면 6 — 통계분석 추천

```
디자인 방향:
  - 상단: 연구 주제 + 변수 설명 입력 폼
  - 결과: 2~3열 카드 그리드 (5~7개)
  - 각 카드:
      - 분석 방법명 (H3, 굵게)
      - 난이도 배지: 초급(초록) / 중급(주황) / 고급(빨강)
      - 적합 이유 (2~3줄 본문)
      - 추천 도구 뱃지: [R] 파랑 / [Python] 금색
      - "직접 실행하기" CTA (Mint 버튼)

UX 포인트:
  - 카드 호버: 테두리 Mint 전환
  - "직접 실행하기" 클릭 → 화면7 해당 탭 자동 선택
  - 카드 5개 미만일 때 빈 카드 대신 여백 처리
```

### 화면 7 — 통계 직접 실행 (R / Python)

```
디자인 방향:
  - 상단 탭: [R] [Python] (Tableau 없음)
  - R 탭: 파랑(#276DC3) 포인트
  - Python 탭: 금색(#F7C948) 포인트
  - 레이아웃: 좌(에디터 60%) / 우(결과+해석 40%)

에디터 영역:
  - Monaco Editor 다크 테마
  - 코드 줄 번호 표시
  - 문법 하이라이팅 (R / Python 각각)

실행 버튼:
  - 크기: 48px 높이, 넉넉한 좌우 패딩
  - cursor: pointer (클릭형 커서 명시)
  - R: 파란 배경 ▶ 실행
  - Python: 금색 배경 ▶ 실행
  - 로딩 중: 스피너 + "실행 중..." 텍스트

결과 패널:
  - 검은 배경, 모노스페이스 폰트
  - 출력 텍스트: R → 초록, Python → 노랑

AI 해석 패널:
  - 카드 형태, 흰 배경
  - 상단: "AI 해석" 라벨 + 스파클 아이콘
  - 로딩: 스켈레톤

초보자 모드 ON:
  - 형광 테두리 단계 가이드 박스
  - 툴팁: Mint 배경 + 화살표
  - 단계 번호: 원형 Mint 배지
  - 사이드 설명 패널: 각 코드 줄 한국어 주석
```
