## 2. 타겟 사용자 & 페르소나

### 페르소나 A — 석사 1학기 김연구 (25세)
- **상황**: AI 관련 주제로 학술지를 써보고 싶지만 어디서 시작해야 할지 모름
- **Pain Point**: 선행연구 검색 방법 모름 / 통계분석 경험 없음 / 영어 논문 읽기 어려움
- **기대**: "내 아이디어가 연구가 될 수 있는지, 어떻게 시작해야 하는지 알고 싶다"

### 페르소나 B — 학부 4학년 이논문 (23세)
- **상황**: 졸업논문 주제를 AI 관련으로 생각 중, 연구모형 개념이 생소
- **Pain Point**: 선행연구와 내 주제의 연결고리를 모름 / 통계 도구 선택 기준 없음
- **기대**: "내 주제에 맞는 선행연구와 연구모형을 한눈에 보고, 통계도 직접 해보고 싶다"

### 페르소나 C — 직장인 연구자 박학술 (34세)
- **상황**: 직장 다니며 학술지 투고 준비 중, 심사 피드백 경험 없음
- **Pain Point**: 논문 완성 후 심사 준비 방법 모름 / 전문가 피드백 받을 곳 없음
- **기대**: "완성한 논문이 심사에서 어떤 피드백을 받을지 미리 연습해보고 싶다"

## 3. 사용자 여정 맵 (User Journey Map)

```
[화면 1] 간편 로그인 (Google / Kakao)
    ↓
[화면 2] 내 생각이 연구로 이어질 수 있을까?
          └─ AI 상담 챗봇 (연동: rq-ai 사이트)
          └─ 연구 가능성 진단 + 주제 구체화
    ↓
[화면 3] 학술지 무엇이 필요할까?
          └─ 학술지 구성요소 단계별 안내 (작성방법)
          └─ AI 전문가 4인 단계별 멘토링
    ↓
[화면 4] 나의 연구주제와 선행연구 연계
          └─ Google Scholar + 국회도서관 + KCI CSV 연동 선행연구 검색
          └─ 피인용 횟수 / 발행연도 / KCI·SCI 등급 표시
          └─ 연구모형 이미지 갤러리 (4~8개 한눈에)
    ↓
[화면 5] 해외 학술지를 한눈에
          └─ Split-View: 좌측 원문 / 우측 번역본 + 하단 요약
    ↓
[화면 6] 통계분석 추천
          └─ 주제별 5~7가지 통계분석 방법 추천
    ↓
[화면 7] 통계 직접 실행
          └─ R / Python 브라우저 내 직접 실행
          └─ 통계 결과 자동 해석 제공
    ↓
[화면 8] 학술지 심사 코칭
          └─ 논문 전체 업로드 → 전문가 4인 피드백 초안
          └─ 가상 교수 3인 심사 실습 (실시간 Q&A)
```

## 5. AI 전문가 패널 시스템 (4인 구성)

> ScholAI의 핵심 차별화 요소

### 5-1. 패널 구성 (v2 — 4인)

| 역할 | 특성 | 주요 담당 영역 |
|------|------|--------------|
| 🎓 논문 다작 박사 | 논문 50편 이상 집필 경험 | 연구 주제 타당성, 논문 구조, 선행연구 연결성 |
| ⚖️ 논문 심사위원 | 실제 심사 경험 풍부 | 연구 방법론, 표본 설계, 이론적 배경, 타당도 |
| 👨‍🏫 논문 지도 교수 | 지도 경험 풍부 | 전반적 논문 완성도, 방향 설정, 격려 코칭 |
| 📊 연구 데이터 분석가 | 계량 통계 전문 | 통계분석 방법 선택, 결과 해석, 데이터 처리 |

### 5-2. 패널 협의 프로세스

```
연구자 질문 입력
    ↓
4인 패널 역할별 독립 의견 생성 (병렬 처리)
    ↓
패널 간 의견 교환 및 합의
    ↓
통합 결과 도출
    ├─ 핵심 통합 의견
    ├─ 패널별 추가 관점 (아코디언 펼침)
    └─ 연구자 선택지 제시 (A안 / B안 / C안)
         ↓
    연구자 선택 → 추가 제안 요청 가능
```

### 5-3. 가상 논문 심사 실습 (교수 3인)

- **시나리오**: 실제 학술지 심사 환경 재현
- **교수 A**: 방법론 중심 심사위원
- **교수 B**: 이론적 배경 중심 심사위원
- **교수 C**: 통계 타당성 중심 심사위원
- **진행**: 3인이 차례로 질문 → 연구자 답변 → AI 즉각 피드백
- **결과**: 심사 후 강점/보완점 리포트 + PDF 다운로드

## 6. 선행연구 검색 기능 상세 (v2)

| 항목 | 내용 |
|------|------|
| 연동 DB 1 | Google Scholar (https://scholar.google.com/) — 해외 논문 중심 |
| 연동 DB 2 | 국회도서관 (https://dl.nanet.go.kr) — 국내 논문/학위논문 |
| 연동 DB 3 | 한국연구재단 KCI 논문정보 CSV (한국연구재단_KCI논문정보_20250825.csv) — 로컬 DB |
| 표시 정보 | 제목 / 저자 / 발행연도 / 피인용 횟수 |
| 등급 표시 | KCI 등재 여부 / SCI·SCIE·SSCI 구분 |
| 요약 | Claude AI 기반 초록 200자 요약 |
| 필터 | 연도 범위 / 논문 유형 / 언어 / 등급 |
| 정렬 | 피인용 횟수순 / 최신순 / 관련도순 |
| 저장 | 논문 즐겨찾기 + 내 연구 폴더 저장 |

### KCI CSV 활용 방식

```
1. 서비스 시작 시 CSV 파일을 PostgreSQL DB에 임포트
2. 키워드 검색 시 DB 쿼리로 빠른 검색 (외부 API 불필요)
3. KCI 등재지 여부, 피인용 횟수 등 정형 데이터로 표시
4. Google Scholar / 국회도서관 결과와 병합 후 중복 제거
```

## 7. 외부 연동 목록 (v2)

| 서비스 | URL | 용도 | 방식 |
|--------|-----|------|------|
| rq-ai | https://rq-ai-620920432884.asia-east1.run.app/ | 연구 주제 상담 | iFrame / postMessage |
| Google Scholar | https://scholar.google.com/ | 해외 선행연구 검색 | Serp API 또는 스크래핑 |
| 국회도서관 | https://dl.nanet.go.kr | 국내 선행연구 검색 | Open API ⚠️ 공개 제한적 (대안: 국립중앙도서관 API 병행) |
| KCI CSV | 한국연구재단_KCI논문정보_20250825.csv | KCI 논문 DB | 로컬 PostgreSQL 임포트 |
| DeepL | - | 영문 논문 번역 | REST API |
| Claude API | - | 상담·패널·심사 AI 엔진 | Anthropic API |
| WebR | - | 브라우저 내 R 실행 | WebAssembly |
| Pyodide | - | 브라우저 내 Python 실행 | WebAssembly |
